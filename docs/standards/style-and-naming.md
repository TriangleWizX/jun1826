# Style And Naming Standards

## Voice

- Public pages: clear, confident, beginner-safe, action-oriented.
- Internal docs: direct, technical, procedural, unambiguous.

## Naming

- Files: lowercase kebab-case.
- URLs: lowercase, short slugs, intent-first wording.
- Event names: snake_case and stable across releases.
- Reusable snippets/partials: noun-purpose pattern (for example: `location-and-reviews`).

## Writing Rules

- Prefer plain language over jargon.
- Explain "why" for non-obvious technical decisions.
- Keep procedures numbered and verifiable.
